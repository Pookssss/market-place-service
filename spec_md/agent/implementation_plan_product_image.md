# แผนการอัปเกรดระบบรูปภาพสินค้าแบบ Dynamic (Multiple Images)

เพื่อยกระดับการนำเสนอสินค้า เราจะปรับปรุงระบบรูปภาพให้เป็นแบบ Dynamic โดยรองรับการอัปโหลดรูปภาพหลายรูปต่อ 1 สินค้า และอาจจะสามารถผูกรูปภาพเข้ากับตัวเลือกสินค้า (Variant) ได้

## 1. Database Schema

จริงๆ แล้วใน `schema_v2.sql` ที่เราทำไว้ ผมได้แอบสร้างตารางเผื่ออนาคตไว้ให้แล้วชื่อว่า **`product_images`** ครับ โครงสร้างคือ:
- `id`, `uuid`
- `product_id` (ผูกกับตารางสินค้า)
- `image_url` (ชื่อไฟล์รูป)
- `is_primary` (เอาไว้บอกว่ารูปไหนเป็นรูปปกหลัก)

*ส่วนในตาราง `product_variants` (ที่สร้างใน Phase 4) ตอนนี้ยังไม่มีฟิลด์สำหรับรูปภาพย่อยครับ*

## 2. API & Logic ที่ต้องปรับแก้

### 2.1 Multer (อัปโหลดไฟล์)
- เปลี่ยนจากการรับไฟล์เดียว `upload.single('image')` เป็นรับหลายไฟล์ `upload.array('images', 10)` (สูงสุดเช่น 10 รูป)

### 2.2 Product Controller (ตอนสร้างสินค้า)
- **POST /api/products**: 
  - จัดการลูปนำชื่อไฟล์ทั้งหมดจาก `req.files` ไปบันทึกลงตาราง `product_images`
  - รูปแรก (Index 0) จะถูกเซ็ตให้เป็น `is_primary = true` (รูปหน้าปก)

### 2.3 Product Model (ตอนดึงข้อมูล)
- ดึงรูปภาพทั้งหมดจากตาราง `product_images` มาแนบเป็น Array ให้กับหน้าบ้าน ตัวอย่างเช่น:
  ```json
  "images": [
      { "id": "uuid-1", "url": "img1.jpg", "is_primary": true },
      { "id": "uuid-2", "url": "img2.jpg", "is_primary": false }
  ]
  ```

## User Review Required

> [!IMPORTANT]
> **มีคำถามเพื่อกำหนดขอบเขตฟีเจอร์นี้ครับ:**
> 
> 1. **Multiple Images:** คุณต้องการแค่อัปโหลด **"รูปภาพหลายรูปรวมกันใน 1 สินค้า"** (โดยเก็บลงตาราง `product_images`) ใช่ไหมครับ?
> 2. **Variant Images (รูปแยกตามสี/ตัวเลือก):** คุณต้องการให้สามารถ **"ผูกรูปภาพเข้ากับ Variant (ตัวเลือก)"** ด้วยไหมครับ? (เช่น เลือกรุ่นสีแดง ก็ให้โชว์รูปสีแดง) 
>    - *ถ้าต้องการข้อ 2 ด้วย ผมจะต้อง ALTER ตาราง `product_variants` เพื่อเพิ่มคอลัมน์ `image_url` เข้าไปด้วยครับ*
> 
> รบกวนแจ้งความต้องการเพิ่มเติม (เช่น เอาทั้งข้อ 1 และข้อ 2) แล้วยืนยัน (Approve) ได้เลยครับ ผมจะลงมือเขียนโค้ดอัปเดตระบบอัปโหลดและการบันทึกรูปภาพทันทีครับ

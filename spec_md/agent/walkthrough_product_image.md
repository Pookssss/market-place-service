# สรุปผลการอัปเกรดระบบรูปภาพสินค้า (Multiple Images)

ตอนนี้ผมได้อัปเกรด API จัดการสินค้า ให้รองรับ **"การอัปโหลดรูปภาพได้หลายรูปใน 1 สินค้า"** เรียบร้อยแล้วครับ! 

ระบบได้เข้าไปเชื่อมต่อกับตาราง `product_images` (ซึ่งเรามีอยู่ใน Database อยู่แล้ว) ทำให้สามารถเก็บรูปได้ไม่จำกัดต่อ 1 สินค้าครับ

## สิ่งที่ได้รับการปรับปรุง

### 1. การอัปโหลดรูป (POST /api/products)
- **เปลี่ยนจาก `image` เป็น `images`**: ตอนยิง API ผ่าน Postman หรือหน้าบ้าน (แบบ `multipart/form-data`) ให้เปลี่ยน Key ฝั่งไฟล์เป็นคำว่า `images` 
- สามารถเลือกไฟล์ส่งมาพร้อมกันได้สูงสุดถึง **10 รูป** 
- **ระบบจัดเก็บฉลาดขึ้น**: 
  - ทุกรูปจะถูกอัปโหลดและสร้างเป็น UUID ของตัวเองลงในตาราง `product_images`
  - รูปแรกที่อัปโหลด (Index 0) จะถูกเซ็ตให้เป็นรูปหน้าปก (`is_primary: true`) อัตโนมัติและเซฟชื่อลงในตารางหลัก (`image_url` ในตาราง `products`) เพื่อให้ระบบเก่าที่ยังไม่ได้อัปเดตยังดึงรูปปกไปโชว์ได้ปกติแบบ Backward Compatibility ครับ

### 2. การดึงข้อมูลรูปภาพ (GET /api/products)
- ทุกครั้งที่คุณดึงข้อมูลสินค้า (ทั้งแบบรายการและแบบรายตัว) ระบบจะแนบ Array `images` กลับมาให้ด้วยครับ

**ตัวอย่าง Response ใหม่:**
```json
{
    "id": "1ffe72d4-2b40-4557-9939-2dbb322bd1c0",
    "name": "T-Shirt",
    "price": "100.00",
    "image_url": "product-123456.jpg", // ยังมีอยู่เพื่อความเข้ากันได้
    "images": [
        {
            "id": "uuid-image-1",
            "url": "product-123456.jpg",
            "is_primary": true
        },
        {
            "id": "uuid-image-2",
            "url": "product-987654.png",
            "is_primary": false
        }
    ],
    "variants": [...]
}
```

---

## 🔥 ขั้นตอนการใช้งาน
1. **Restart Node Server** (`Ctrl + C` แล้วรัน `node server.js` หรือ `npm run dev`) เพื่ออัปเดตโค้ดล่าสุดเข้า Memory
2. **ไม่ต้องรัน SQL ใหม่:** เพราะตาราง `product_images` ถูกสร้างไว้ตั้งแต่ตอนรัน `schema_v2.sql` ครั้งแรกสุดแล้วครับ
3. ลองเทสยิง Postman สร้างสินค้าใหม่ โดยอัปโหลดไฟล์ใน Field ที่ชื่อว่า `images` (สามารถกดเลือกหลายไฟล์ได้) แล้วดู Response ที่ได้ครับ!
